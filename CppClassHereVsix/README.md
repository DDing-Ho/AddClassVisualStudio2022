# CppClassHereVsix

`CppClassHereVsix`는 Visual Studio 2022에서 C++ 클래스를 더 편하게 만들 수 있게 해주는 확장 프로그램입니다.

기본 `추가 -> 클래스` 기능은 솔루션 탐색기에서 특정 폴더나 필터를 우클릭해도 실제 `.h`, `.cpp` 파일이 프로젝트 루트에 생성되는 경우가 많습니다. 이 확장은 그 불편함을 줄이기 위해 `Add C++ Class Here...` 명령을 추가하고, 사용자가 선택한 위치를 기준으로 파일을 만들고 프로젝트에 바로 추가해줍니다.

## 이 확장을 설치하면 무엇이 추가되나요?

확장을 설치하면 아래 위치에 `Add C++ Class Here...` 메뉴가 추가됩니다.

- 솔루션 탐색기에서 프로젝트, 폴더, 필터를 우클릭했을 때의 `추가` 메뉴
- 상단 메뉴의 `도구` 메뉴

이 메뉴를 누르면 기본 Visual Studio 클래스 추가 창 대신, 이 확장에서 제공하는 전용 클래스 생성 다이얼로그가 열립니다.

## 이 확장은 어떤 문제를 해결하나요?

가장 큰 목적은 `클래스를 어디에 생성할지`를 더 예측 가능하게 만드는 것입니다.

기본 클래스 추가 기능은 솔루션 탐색기에서 폴더를 선택했더라도 실제 디스크에는 프로젝트 루트에 파일을 만드는 경우가 있습니다. 이 확장은 다음처럼 동작을 명확하게 나눕니다.

- 실제 폴더를 선택하면: 그 폴더에 파일을 만듭니다.
- 프로젝트 루트를 선택하면: 프로젝트 루트에 파일을 만듭니다.
- VC 필터를 선택하면: 실제 디스크에는 프로젝트 루트에 파일을 만들고, 프로젝트 안에서는 선택한 필터 아래에 추가합니다.

## 한눈에 보는 동작

### 실제 폴더를 우클릭한 경우

이 경우가 가장 직관적인 동작입니다.

- 실제 생성 위치: 우클릭한 실제 디스크 폴더
- 프로젝트 표시 위치: 선택한 그 폴더 아래

예:

- `Source\Gameplay` 폴더를 우클릭합니다.
- `Player` 클래스를 생성합니다.
- 실제 파일은 `Source\Gameplay\Player.h`, `Source\Gameplay\Player.cpp`에 생성됩니다.
- 솔루션 탐색기에서도 `Gameplay` 폴더 아래에 추가됩니다.

### VC 필터를 우클릭한 경우

이 부분이 가장 헷갈리기 쉬운 부분입니다.

VC 필터는 실제 디스크 폴더가 아니라 Visual Studio 안의 가상 분류이기 때문에, 이 확장은 실제 파일을 프로젝트 루트에 생성하고, 생성한 파일을 선택한 필터 아래에 추가합니다.

- 실제 생성 위치: 프로젝트 루트 디렉터리
- 프로젝트 표시 위치: 선택한 필터 아래

예:

- `Gameplay`라는 VC 필터를 우클릭합니다.
- `Player` 클래스를 생성합니다.
- 실제 파일은 `<프로젝트 루트>\Player.h`, `<프로젝트 루트>\Player.cpp`에 생성됩니다.
- 솔루션 탐색기에서는 `Gameplay` 필터 아래에 보입니다.

즉, 필터를 선택했다고 해서 실제 디스크에 `Gameplay` 폴더가 생기거나 그 안에 파일이 생성되지는 않습니다.

### 프로젝트 루트를 우클릭한 경우

- 실제 생성 위치: 프로젝트 루트 디렉터리
- 프로젝트 표시 위치: 프로젝트 루트

## 어떤 프로젝트에서 사용할 수 있나요?

이 확장은 아래 조건에서만 동작합니다.

- Visual Studio 2022
- `vcxproj` 기반 C++ 프로젝트

아래 경우는 지원하지 않습니다.

- C# 같은 비 `vcxproj` 프로젝트
- CMake / Open Folder 프로젝트
- 파일 노드를 직접 우클릭한 경우
- 다중 선택 상태
- 현재 선택 위치를 실제 프로젝트 위치로 해석할 수 없는 경우

지원하지 않는 상황에서 실행하면 오류 메시지가 표시되고 파일은 생성되지 않습니다.

## 클래스 생성 창에서는 무엇을 입력하나요?

이 확장은 자체 다이얼로그를 사용합니다. 기본적으로 아래 항목을 입력할 수 있습니다.

- 클래스 이름
- `.h` 파일 이름
- `.cpp` 파일 이름
- 기본 클래스
- 액세스 지정자
- `인라인` 옵션

## 다이얼로그는 어떻게 동작하나요?

### 클래스 이름과 파일 이름 자동 맞춤

처음에는 클래스 이름을 입력하면 파일 이름도 자동으로 같이 맞춰집니다.

예:

- 클래스 이름에 `Player`를 입력하면
- `.h` 파일은 `Player.h`
- `.cpp` 파일은 `Player.cpp`

으로 자동 입력됩니다.

사용자가 `.h` 또는 `.cpp` 파일명을 직접 수정하면 자동 맞춤이 내부적으로 해제됩니다. 이후 클래스 이름을 다시 수정하면 자동 맞춤이 다시 활성화되어 파일명이 클래스 이름 기준으로 다시 바뀝니다.

### `인라인` 옵션

`인라인` 옵션을 체크하면 `.cpp` 파일을 만들지 않고 헤더 파일만 만듭니다.

이때는:

- `.cpp` 파일 입력칸이 비활성화됩니다.
- 실제로는 `.h` 파일만 생성됩니다.
- 프로젝트에도 헤더 파일만 추가됩니다.

## 어떤 코드가 생성되나요?

### 기본 클래스가 없는 경우

예를 들어 `Player`를 만들면 헤더 파일은 이런 형태로 만들어집니다.

```cpp
#pragma once

class Player
{
};
```

소스 파일은 이런 식으로 생성됩니다.

```cpp
#include "Player.h"
```

### 기본 클래스를 입력한 경우

예를 들어:

- 클래스 이름: `Player`
- 기본 클래스: `Actor`
- 액세스: `public`

으로 입력하면 헤더는 이런 식으로 생성됩니다.

```cpp
#pragma once

class Player : public Actor
{
};
```

현재 버전에서는 생성자/소멸자 자동 생성 기능은 지원하지 않습니다.

## 파일 이름에는 어떤 규칙이 있나요?

- 파일 이름에는 경로를 넣을 수 없습니다.
- 잘못된 파일명 문자는 허용되지 않습니다.
- 확장자를 직접 입력하지 않으면 `.h`, `.cpp`가 자동으로 붙습니다.
- 헤더 파일명과 소스 파일명은 서로 같을 수 없습니다.
- 같은 이름의 파일이 이미 있으면 생성이 실패합니다.

## 파일을 만든 뒤에는 어떻게 되나요?

파일을 만들면 확장은 그 파일을 프로젝트에 즉시 추가합니다.

- 실제 폴더를 선택했다면: 그 폴더 아래에 추가됩니다.
- VC 필터를 선택했다면: 그 필터 아래에 추가됩니다.
- 프로젝트 루트를 선택했다면: 프로젝트 루트에 추가됩니다.

그리고 헤더 파일을 자동으로 열어줍니다.

## 언어와 테마는 어떻게 동작하나요?

### UI 언어

Visual Studio UI 언어를 따라 아래 항목이 한국어/영어로 바뀝니다.

- 메뉴 이름
- 다이얼로그 라벨
- 오류 메시지
- 확장 프로그램 이름과 설명

### 테마

다이얼로그는 Visual Studio 현재 테마 색을 읽어 다음 요소에 반영합니다.

- 창 배경
- 입력 컨트롤 배경
- 텍스트 색
- 버튼 색
- 테두리 색

## 설치 방법

빌드 없이 바로 설치하려면 아래 파일을 실행하면 됩니다.

- `Installer\CppClassHereVsix-1.3.6.vsix`

설치 순서:

1. Visual Studio 2022를 종료합니다.
2. `Installer\CppClassHereVsix-1.3.6.vsix`를 실행합니다.
3. 설치가 끝나면 Visual Studio를 다시 실행합니다.

## 저장소에서 무엇을 보면 되나요?

- `Installer/`: 바로 설치 가능한 VSIX 파일이 들어 있습니다.
- `CppClassHereVsix/`: 확장 프로그램 소스 코드가 들어 있습니다.
- `CppClassHereVsix/ScreenShot/`: UI 비교용 참고 이미지가 들어 있습니다.
- `CppClassHereVsix.sln`: 솔루션 파일입니다.

## 소스를 직접 빌드하려면 무엇이 필요한가요?

소스를 수정하거나 새 버전을 빌드하려면 Visual Studio 2022 SDK가 필요합니다.

```powershell
& "C:\Program Files\Microsoft Visual Studio\2022\Community\MSBuild\Current\Bin\MSBuild.exe" .\CppClassHereVsix.sln /p:Configuration=Release /t:Build /m:1 /nologo /v:minimal
```

빌드 결과는 보통 아래 경로에 생성됩니다.

- `CppClassHereVsix\bin\Release\CppClassHereVsix.vsix`
- `CppClassHereVsix\bin\Release\CppClassHereVsix-1.3.6.vsix`
